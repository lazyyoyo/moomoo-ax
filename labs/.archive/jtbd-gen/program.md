# JTBD Discovery 규칙

## 역할
seed.md를 입력받아 JTBD 분석 문서(jtbd.md)를 생성하는 스크립트를 작성/개선한다.

## jtbd.md 구조
- Core Job Statement: "When [상황], I want to [동기], so I can [기대 결과]."
- Job Map: 정의하기→찾기→준비하기→실행하기→확인하기→수정하기
- Competing Solutions: 대안별 잘하는 점 / 못하는 점
- Underserved Needs: 높은 중요도 + 낮은 만족도인 니즈

## 제약
- 마크다운 형식
- Competing Solutions은 최소 2개
- 입력에 없는 정보는 합리적 추론만 (지어내지 않음)

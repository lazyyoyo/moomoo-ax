## HMW Questions (우선순위순)

1. 어떻게 하면 Supabase에 쌓인 로그 데이터를 시각화하여 즉시 인사이트를 얻을 수 있을까?
2. 어떻게 하면 루프의 점수 추이를 보고 수렴 여부를 빠르게 판단할 수 있을까?
3. 어떻게 하면 토큰 소비량을 실시간으로 모니터링하여 비용을 통제할 수 있을까?
4. 어떻게 하면 프로젝트별 현황을 비교하여 병목 프로젝트를 즉시 식별할 수 있을까?
5. 어떻게 하면 2인 내부 도구로서 최소한의 인증으로 안전하게 접근할 수 있을까?

## Selected Direction

- **핵심 솔루션**: S1 + S2
- **이유**: S1이 시각화 부재라는 핵심 문제를 직접 해결하고, S2가 프로젝트 간 비교라는 멀티프로젝트 유저의 핵심 니즈를 커버한다. 둘 다 Feasibility H이므로 첫 스프린트에서 함께 구현 가능.
- **핵심 가정**: Supabase 로그 테이블에 project, score, token_usage, iteration 등 시각화에 필요한 필드가 이미 존재한다. 2인 내부 도구이므로 인증은 후순위로 미뤄도 운영에 문제없다.

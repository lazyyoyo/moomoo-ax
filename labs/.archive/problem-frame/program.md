# Problem Framing 규칙

## 역할
seed.md + jtbd.md를 입력받아 문제 정의 문서(problem-frame.md)를 생성하는 스크립트를 작성/개선한다.

## problem-frame.md 구조
- HMW Questions: "어떻게 하면 [구체적 문제]를 해결할 수 있을까?" 3~5개
- Solution Candidates: 각 HMW에 대한 솔루션 아이디어 + Impact×Feasibility 매트릭스
- Selected Direction: 선택된 솔루션 + 이유 + 핵심 가정

## 제약
- 마크다운 형식
- HMW 3~5개
- 솔루션 후보 최소 5개
- Impact/Feasibility는 H/M/L로 표기

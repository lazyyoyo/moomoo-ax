## Core Job Statement
When 바이브 코딩으로 빠르게 제품을 만들 수 있지만 품질 리뷰와 피드백 재작업 루프가 오너 시간을 가장 많이 잡아먹고, 프로젝트마다 CPS·PRD 작성 같은 동일한 시행착오가 반복될 때, I want to 평가 기준이 없어 "충분히 좋은가" 판단이 매번 오너의 감에 의존하는 문제를 해결하고 싶다, so I can team moomoo(yoyo+jojo)의 IT 제품 제작 과정에서 단계별 산출물(CPS/PRD/디자인/코드) 생성 스크립트를 auto-research 루프로 자동 개선하고, 검증된 스크립트를 plugin 스킬로 승격해 다른 프로젝트에 재사용하면서 자체 진화하는 메타 자동화 엔진을 갖출 수 있다.

## Job Map
1. **정의하기**: 오너가 program.md(규칙)와 rubric.yml(평가 기준)을 정의해 단계별 산출물(CPS/PRD/디자인/코드) 생성의 기준을 세운다.
2. **찾기**: labs/에서 define 파이프라인 5단계(seed→jtbd→problem-frame→scope→prd) 중 개선이 필요한 스크립트를 찾는다.
3. **준비하기**: 범용 루프 엔진이 program.md + rubric.yml을 로드하고 subprocess.run 기반 Claude 호출 환경을 준비한다.
4. **실행하기**: auto-research 루프로 단계별 산출물 생성 스크립트를 자동 개선하고 Supabase 로그에 기록한다.
5. **확인하기**: rubric.yml 평가 기준으로 "충분히 좋은가"를 판단하고 Vercel 대시보드에서 결과를 확인한다.
6. **수정하기**: 검증된 스크립트를 labs/에서 plugin/ 스킬로 승격해 다른 프로젝트에 재사용하며 자체 진화시킨다.

## Core Job Statement
When 로그가 Supabase에 쌓이기만 하고 시각화가 없어서, 루프가 수렴하는지·토큰을 얼마나 쓰는지·어떤 프로젝트가 병목인지 알 수 없을 때, I want to ax-loop 실행 로그(Supabase)를 Vercel 대시보드로 시각화하여, so I can 점수 추이·토큰 소비·프로젝트별 현황을 한눈에 파악한다.

## Job Map
1. **정의하기**: 여러 프로젝트(rubato, rofan-world, moomoo-ax 등)에서 모니터링해야 할 루프 품질과 비용 항목을 정한다.
2. **찾기**: Supabase에 쌓인 ax-loop 실행 로그에서 점수 추이·토큰 소비·프로젝트별 현황 데이터를 찾는다.
3. **준비하기**: Supabase 데이터 소스를 Vercel + Next.js 대시보드에 연결하여 시각화를 준비한다.
4. **실행하기**: 대시보드로 시각화하여 루프가 수렴하는지·토큰을 얼마나 쓰는지·어떤 프로젝트가 병목인지 확인한다.
5. **확인하기**: 점수 추이·토큰 소비·프로젝트별 현황을 한눈에 파악하여 데이터에서 인사이트를 얻는다.
6. **수정하기**: 병목인 프로젝트를 식별하고 루프 품질과 비용 모니터링 결과에 따라 조치한다.

## Competing Solutions

| 대안 | 잘하는 점 | 못하는 점 |
|------|----------|----------|
| Supabase에 로그가 쌓이는 현재 방식 | 데이터는 있다 | 시각화가 없어서 인사이트가 없는 상태. 루프가 수렴하는지·토큰을 얼마나 쓰는지·어떤 프로젝트가 병목인지 알 수 없다 |
| 시각화 미사용 (수작업 확인) | 시드에서 확인 불가 | 점수 추이·토큰 소비·프로젝트별 현황을 한눈에 파악할 수 없다 |

## Underserved Needs
- **시각화**: 데이터는 있는데 인사이트가 없는 상태 — 시각화 부재가 핵심 미충족 니즈
- **점수 추이 파악**: 루프가 수렴하는지 알 수 없다
- **토큰 소비 모니터링**: 토큰을 얼마나 쓰는지 알 수 없다
- **프로젝트별 현황 비교**: 어떤 프로젝트가 병목인지 알 수 없다

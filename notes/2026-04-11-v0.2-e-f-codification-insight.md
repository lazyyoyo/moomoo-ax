### 1. skill invoke 메커니즘 괴리

**product loop** (실제 team-ax 가 돌 때, Claude Code plugin 으로):
- 오너: `/ax-implement`
- Claude Code 런타임이 SKILL.md 를 skill 로 load
- Claude 에이전트가 interactive loop 안에서 Bash/Read/Write tool 을 여러 번 호출
- 자연어 부분은 Claude 가 판단, 실행 부분은 tool 로 deterministic 수행

**levelup loop** (현재 moomoo-ax script.py):
- `loop.py → script.py → claude -p "SKILL.md 전체 + fixture"` one-shot
- Claude 는 SKILL.md 를 읽어서 **상상** 으로 mental 수행
- Bash/Read/Write 같은 tool 접근 없음
- SKILL.md 안에 `[run: foo.sh]` 같은 블록이 있어도 실제로 안 돌음

→ levelup loop 가 측정하는 품질 ≠ product loop 의 실 품질. 측정 괴리.

### 2. `[run: ...]` 규약 부재 → "자연어 압축" 이 codification 으로 착각됨

이번 iter 2 의 SKILL.md 는 295줄 → 54줄로 줄었고 HARD RULES R-LEN/R-DRY 섹션이 추가됐다. 겉으로는 "결정적 규칙화" 로 보이지만 내용물은 **여전히 자연어**:

```markdown
### R-LEN. 공개 함수 본체 ≤ 60줄
- 모듈에서 export 되는 모든 함수의 본체는 60줄을 넘을 수 없다
- 60줄을 넘길 것 같으면 즉시 내부 블록을 function 으로 분리한다
...
```

진짜 codification 은 이걸 **실제 script** 로 뽑고 SKILL.md 가 이렇게 적는 것:

```markdown
### R-LEN
[check: scripts/check_r_len.py --max-lines 60 {target}]
```

그러면 실 script 가 AST 파싱으로 결정적으로 체크하고, SKILL.md 는 "이 체크를 실행해" 만 말함. Claude 가 재해석할 여지 없음.

이번 run 은 "자연어 → 자연어(축약+강조)" 까지만 했다. **codification 0 단계**.

### 3. improve_target 이 single file

현재 `improve_target: ../../plugin/skills/ax-implement/SKILL.md` — **파일 1개**. 하지만 오너 지적대로, skill 의 수정 범위는:

- `SKILL.md` 본문
- `references/*.md` (현재 로드조차 안 되는 장식)
- `scripts/**/*.py` (v0.3+ 에서 추출될 실 script)
- `agents/**/*.md` (v0.4+ 에서 도입될 subagent 정의)

이 **모두** 가 하나의 skill bundle 로 묶여 같이 개선돼야 한다. single-file improve_target 은 v0.2 파이프 검증용 임시 구조.

### 본질

이건 ax-qa 고유의 문제가 아니라 **levelup loop 전체의 실행 환경 한계**:

> `subprocess.run(["claude", "-p", prompt])` 는 Claude Code 의 tool loop 이 아니라 single-shot LLM 호출이다.

Claude Code 런타임 안에서만 되는 것 (현재 `claude -p` 에서 불가):
- Bash/Read/Write tool call
- MCP server (Playwright, Supabase 등)
- Agent tool (서브에이전트 호출)
- Skill load / `/command` 호출
- interactive tool loop (여러 단계 계획 → 실행 → 관찰 반복)

ax-implement 도 같은 한계에 막혔지만 "산출물이 코드 텍스트" 라 one-shot 으로 근사 가능했음. ax-qa 는 "실 실행 환경" 이 필수라 근사 불가.

### 오너의 과거 경험과 동일한 벽

my-agent-office 에 Ralph loop 을 Python script 로 넣으려다 제약 많아 skill 로 풀었던 경험과 같은 벽. moomoo-ax 가 "Python 만으로 가능" 가정으로 설계된 v0.1~v0.2 의 전제가 깨진 지점.

## v0.3 설계의 방향 (리서치 후 확정)

**하이브리드가 유력**:

접합:


이게 가능하면 두 세계의 장점이 합쳐짐. 가능하지 않으면 v0.3 은 "완전 skill 기반 loop" 로 재설계 (Python 은 얇은 entry point 로 축소).

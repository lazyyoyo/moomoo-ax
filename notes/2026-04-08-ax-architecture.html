루프 2 내부 — 코드 vs AI 영역 분리

핵심 설계
단계 실행 주체 내용 실행 방식
CPS 입력 오너 무엇을 만들지 정의 — 유일한 오너 개입 수동 (문서 작성)
페르소나 도출 AI CPS 컨텍스트에서 사용자 유형 자동 생성 claude -p → JSON
평가 기준 생성 AI 고정 루브릭 + 동적 루브릭 (Yes/No 체크리스트) claude -p → YAML
디자인/구현 AI 워커 Claude(설계) + Codex(코드) + Gemini(UI/비전) headless CLI → JSON
① 정적 게이트 코드 lint, typecheck, format pnpm lint (exit code)
② 시각 게이트 코드 Playwright 스크린샷 diff toHaveScreenshot()
③ 구조 게이트 코드 ARIA 스냅샷 비교 ariaSnapshot()
④ Judge 게이트 AI + 코드 체크리스트 Yes/No → 합성 점수 (코드로 계산) gemini -p → JSON → 스크립트
판정 코드 점수 비교 → keep / discard / crash if score > best → commit, else reset
재작업 AI 실패 항목 → 개선 지시 프롬프트 자동 생성 claude -p → 다음 iteration 입력
로깅 코드 iteration_N.json + 토큰 사용량 기록 스크립트 자동 기록

ax-build flow (v0.8)

worktree 제거 + Codex 워커 N개 + 파일 whitelist 격리 + 단일 브랜치
lead (Claude main) 에이전트 (planner/executor) 오너 게이트 Codex 워커 (one-shot) 분기/에러 시작/종료
Phase 1 — 진입 + 환경 검증
0 ax-build 진입 lead
오너가 /ax-build 호출. 현재 버전 / scope / plan 입력 컨텍스트 확인.
↓
1 precheck — 환경 검증 lead
  • codex --version + 로그인 확인
  • tmux 안에서 기동 중인지 (중첩이면 현재 세션 사용)
  • git worktree list 정리 상태
실패 → [중단 + 설치/설정 안내 출력]
↓
Phase 2 — 계획 + 분할
2 planner — 태스크 분할 planner agent
scope + spec 읽고 태스크를 파일 집합 단위로 분할. 산출: .ax/plan.json
  • 태스크 간 files 겹침 없음 → 병렬 후보
  • 겹침/논리 의존 → blockedBy 선언 (순차)
  • 공유 파일(타입 등) → "공통 기반" 태스크로 별도 분리 → 최우선
↓
3 오너 게이트 — plan 승인 owner
.ax/plan.json 분할 결과 검토.
  • 승인 → Phase 3으로
  • 반려 → planner 재분할 (2번으로)
초기엔 수동 게이트 필수. 분할 품질이 병렬 효과 좌우.
↓
Phase 3 — 공통 기반 (순차)
4 공통 기반 태스크 실행 codex × 1 (순차)
공유 파일(타입 정의 등) 변경은 단일 워커가 순차 처리 → 이후 병렬 태스크의 기반.

codex ... "/ax-execute .ax/workers/common/inbox.md"
↓
5 공통 기반 커밋 lead
result.json 확인 → git status whitelist 대조 → lead 일괄 커밋.
실패 시 [중단 + 오너 보고]
↓
Phase 4 — 병렬 라운드 (blockedBy 해소 시 반복)
6 병렬 가능 태스크 선정 lead
.ax/plan.json에서 blockedBy 빈 태스크 집합 추출 → 이번 라운드 병렬 대상.
↓
7 워커 인프라 준비 lead
  • tmux 윈도우 ax-workers (없으면 생성, split tiled)
  • 워커별 .ax/workers/<id>/inbox.md 기록 (과제 + whitelist + result.json 경로)
↓ tmux pane × N 동시 스폰
← 병렬 실행 (최대 5 워커) →
8a Worker 1 codex
/ax-execute .ax/workers/T1/inbox.md

ax-execute 스킬이 프로토콜 담당 (whitelist / TDD / result.json / self-check).
8b Worker 2 codex
/ax-execute .ax/workers/T2/inbox.md
8c Worker 3 codex
/ax-execute .ax/workers/T3/inbox.md
↓
9 폴링 + 수렴 감지 lead
10초 간격으로 .ax/workers/*/result.json 읽기.
statusline에 workers: 2 done / 1 in-progress 형태로 요약.
↓ 분기
✓ all done
10번으로
✗ error
즉시 중단 + 오너 보고
⏱ timeout (30분)
kill-pane + 순차 재시도 옵션
⏸ blocked
notes 기반 오너 개입
↓
10 lead 검증 + 일괄 커밋 lead
  • result.json.files_touched 집계
  • git status --porcelain 대조 → whitelist 밖 변경 탐지 (2중 가드)
  • 태스크 단위 커밋 (워커별 1개 or 논리 묶음)
  • 커밋 메시지: 태스크 ID + 제목 + 주요 변경 (한글)
↓
11 plan.json 업데이트 lead
완료 태스크 표시. 다른 태스크의 blockedBy에서 해제.
↓ 분기
남은 태스크 있음
→ 6번으로 루프 (다음 병렬 라운드)
모두 완료
→ Phase 5
↓
Phase 5 — 마무리
12 tmux 정리 + .ax 아카이브 lead
ax-workers 윈도우 닫기. .ax/ 결과물은 로그로 아카이브(선택).
↓
13 ax-qa로 전달 next skill
단일 브랜치 version/vX.Y.Z가 커밋 완료 상태 → ax-qa 진입.

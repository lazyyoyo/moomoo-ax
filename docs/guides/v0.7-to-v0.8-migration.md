# v0.7 → v0.8 마이그레이션 가이드

team-ax v0.8은 ax-build 병렬 엔진을 재설계한다. **사용자 관점의 변경점**과 **행동 변화**를 정리.

## 요약

- worktree가 사라진다. 단일 브랜치 위에서 **파일 whitelist 격리**.
- ax-build 워커는 전부 **codex**. 기존 `executor.engine=claude` 토글 제거.
- 워커 각자 커밋 → **lead가 일괄 커밋**.
- tmux pane split grid(`ax-workers`)로 모든 워커를 **한 윈도우에서 관찰**.

## 업그레이드 절차

1. team-ax 플러그인 업데이트 (marketplace에서 v0.8.x 설치)
2. `/ax-codex install` 재실행 — codex의 ax-execute 스킬을 v0.8로 갱신
3. 프로젝트 `.claude/settings.json`에서 `executor.engine` 키는 **삭제 또는 방치** (v0.8은 무시)
4. tmux 안에서 claude 기동 전제 확인 — `echo $TMUX` 비어있지 않아야 함
5. 시험 `/ax-build` 1회 — precheck 통과 후 plan 단계부터 정상 진행 확인

## 새 사전 점검

```bash
bash plugin/scripts/ax-build-orchestrator.sh precheck
```

체크 항목:
- tmux 안에서 실행 중
- `codex --version` OK
- `codex login status` (지원 버전이면) OK
- git 리포
- `~/.codex/skills/ax-execute/` 설치

하나라도 실패하면 안내 메시지대로 수정 후 재시도.

## 사라진 것 / 대체물

| v0.7 | v0.8 |
|---|---|
| `.claude/worktrees/<name>/` 디렉토리 | 없음 (단일 브랜치) |
| `version/vX.Y.Z-<name>` 워커 브랜치 | 없음 |
| `git worktree add/remove` | 없음 |
| `executor.engine: claude\|codex` | codex 고정 |
| `.ax-brief.md` (공유 지시서) | `.ax/workers/<task_id>/inbox.md` (워커별) |
| `.ax-status` (building/review-ready/needs-fix/merge-ready) | `.ax/workers/<task_id>/result.json` (done/blocked/error) |
| stdout `DONE` / `BLOCKED: ...` 계약 | result.json 파일 스키마 |
| 워커 태스크 단위 커밋 | lead 일괄 커밋 |
| 포트별 dev server / review-ready 오너 확인 | 병렬 단계는 코드만. dev server는 4단계(오너 최종 확인) 단계에서 lead가 기본 포트로 |
| `--allow` / `--block` 인자 (ax-execute) | inbox.md 안의 whitelist로 통합 |
| `ax-build-orchestrator.sh worktree/merge` 서브커맨드 | `prepare-window/spawn/cleanup` 서브커맨드 |
| `planner` 에이전트가 worktree 전략 결정 | `planner`가 파일 집합 단위 분할 + `blockedBy` 선언 |

## 새로 생긴 것

- `.ax/plan.json` — planner 산출 SSOT. lead 오케스트레이터가 이 파일을 보고 스폰·폴링·커밋
- `.ax/workers/<task_id>/inbox.md` — 워커별 과제 지시 (템플릿: `plugin/skills/ax-build/templates/worker-inbox.md.tmpl`)
- `.ax/workers/<task_id>/result.json` — 워커 산출 (스키마 고정)
- tmux 윈도우 `ax-workers` — 모든 워커 pane을 tiled grid로 배치
- `ax-build-orchestrator.sh precheck / prepare-window / spawn / status / cleanup`

## `.gitignore` 권장

`.ax/` 디렉토리는 중간 산출이므로 프로젝트 `.gitignore`에 추가 권장:

```
.ax/
```

필요 시 CI에서 아카이브하고 싶으면 `.ax/plan.json`만 예외로.

## v0.7 흔적 정리

- 기존 `.claude/worktrees/` 디렉토리 — 더 이상 사용되지 않음. 안에 `.git` 링크만 있다면 `git worktree remove`로 정리하거나 무시(다음 ax-build에서 touch 하지 않음). **휴지통 이동**(`mv ~/.Trash/`)이 프로젝트 규칙
- `.ax-brief.md` / `.ax-status` 파일 — 레거시 산출. 안전하게 `mv ~/.Trash/`
- `executor.engine` 설정 — `.claude/settings.json`에 있으면 v0.8이 무시. 지워도 무방
- `plugin/agents/executor.md` — Claude 레거시 executor. v0.8에선 자동 호출 경로 없음. 수동 호출이 필요한 경우에만 의미

## 행동 변화 (오너 관점)

### 기존

1. `/ax-build` → planner가 worktree 전략 제안
2. build-plan 승인 → orchestrator가 워크트리 + tmux window N개 생성
3. 오너가 tmux window를 하나씩 전환하며 각 워커 진행 관찰
4. 각 워커가 review-ready → 포트별 dev server로 확인 → merge-ready
5. lead가 worktree 브랜치 순차 머지
6. ax-qa 전달

### v0.8

1. `/ax-build` → precheck → planner가 **파일 집합 단위** 분할 제안 (`.ax/plan.json`)
2. plan 승인 → orchestrator가 `ax-workers` 윈도우에 **pane tiled grid**로 codex 워커 스폰
3. 오너가 `Ctrl-b w`로 `ax-workers` 윈도우 이동, **한 화면에 모든 워커 관찰**
4. 모든 워커 `result.json done` → **lead가 whitelist 대조 + 일괄 커밋**
5. 남은 태스크 있으면 다음 병렬 라운드 (2~4 반복)
6. 모두 완료 → lead가 기본 포트로 통합 dev server → 오너 최종 확인 → ax-qa 전달

병렬 워커 개수는 라운드별 plan 결과에 따라 달라짐 (기본 2-3, 최대 5).

## 트러블슈팅

### codex 미설치/미로그인

```
✗ codex CLI: 미설치
  → npm install -g @openai/codex
```

또는 `codex login`.

### tmux 밖 실행

```
✗ tmux: 세션 밖에서 실행 중
  → tmux new-session -s ax  → 그 안에서 claude 시작
```

### ax-execute 스킬 미설치

```
✗ codex skill ax-execute: 미설치
  → /ax-codex install
```

### 워커가 whitelist 밖 파일을 수정했다고 error

워커 ax-execute가 `git status --porcelain`로 self-check한 결과. **임의 revert는 금지**. 오너가 직접 확인하고:

- 수정이 정당했으면 inbox.md의 whitelist 갱신 후 재스폰
- 정말 잘못이면 수동 `git checkout --` 또는 개별 대응

### pane TUI가 깨져 보임

워커 5개 이상이면 pane 폭이 좁아져 codex TUI 렌더가 이상할 수 있음. tmux 창 크기를 늘리거나 plan 분할을 조정.

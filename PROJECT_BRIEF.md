# moomoo-ax — Project Brief

## 한 줄 요약

auto-research 루프로 업무 산출물 품질을 자동 개선하는 엔진.

## 무엇을 만드는가

단일 업무(PRD 작성, 디자인, 구현 등)에 대해 "생성 → 평가 → 개선" 루프를 자동으로 돌려서, 오너 개입 없이 산출물 품질을 올리는 시스템.

## 왜 만드는가

- 바이브 코딩으로 빠르게 만들 수 있지만, 품질은 오너가 직접 리뷰해야 함
- 리뷰 → 피드백 → 재작업 루프가 오너 시간을 가장 많이 잡아먹음
- "평가 기준을 정의하고 루프를 돌리면" 오너 개입 없이 품질이 수렴할 수 있음

## 핵심 원리 (auto-research에서)

1. **수정 범위를 좁힌다** — 한 번에 한 파일/문서만 수정
2. **eval이 숫자다** — 체크리스트 → 합산 점수. gradient가 있어야 개선 방향을 알 수 있음
3. **반복이 빠르다** — 문서 생성 ~30초, 코드 생성 ~1분
4. **하네스가 불변** — 평가 기준은 루프 안에서 안 변함. 변경은 meta-loop에서

## 누가 쓰는가

- **yoyo** — rubato, dashboard 등 프로젝트
- **남편** — rofan-world 등 프로젝트
- 두 사람의 프로젝트 진행 상황, 토큰 소비량을 한 곳에서 추적

## 첫 프로젝트 (dogfooding)

moomoo-ax 자체의 대시보드를 만드는 것.
- v0.2에서 대시보드 PRD를 루프로 생성
- v0.3~에서 대시보드 코드를 루프로 생성
- 만들면서 루프 자체를 강화

## 기술 스택

- **오케스트레이터**: Python (loop.py)
- **워커**: Claude CLI (`claude -p` subprocess)
- **eval**: LLM Judge (rubric.yml → Yes/No → 합산 점수)
- **로그**: JSON 파일 → (v0.4~) Supabase
- **대시보드**: (v0.4~) Vercel

## 성공 기준 (v1.0)

- [ ] 3가지 이상 업무 유형에서 루프가 수렴함 (PRD, 디자인, 구현)
- [ ] 오너 개입 없이 80% 이상 산출물이 기대치 도달
- [ ] yoyo + 남편 프로젝트에서 실사용

# 리뷰 체크리스트

pr-review-toolkit 기반. reviewer가 전체 변경사항 리뷰 시 사용.

## 체크 항목

### 1. Spec 정합성
- [ ] 각 spec 시나리오가 구현됨
- [ ] 완료 기준 충족
- [ ] 누락된 시나리오 없음

### 2. 디자인 시스템 준수
- [ ] DESIGN_SYSTEM.md 토큰만 사용 (하드코딩 없음)
- [ ] 확정 컴포넌트 목록 참조
- [ ] 토큰/컴포넌트 반복 패턴 감지 (3회 이상 → 오너 보고)

### 3. Silent Failure 검출
- [ ] 빈 catch 블록 없음
- [ ] 에러 무시 없음 (catch 후 아무 처리 안 함)
- [ ] 에러 로깅 있음

### 4. 타입 설계
- [ ] DB/API/UI 타입 분리
- [ ] 불변성 적절히 사용
- [ ] any 타입 최소화

### 5. 테스트 커버리지
- [ ] 핵심 경로 테스트 존재
- [ ] 엣지 케이스 테스트
- [ ] 에러 경로 테스트

### 6. 보안
- [ ] 하드코딩된 키/토큰/시크릿 없음
- [ ] 민감정보 로그 출력 없음
- [ ] env 파일 직접 읽기 없음

### 7. 텍스트 하드코딩
- [ ] 모든 UI 텍스트가 i18n/copy 경유
- [ ] 컴포넌트에 직접 문자열 없음

### 8. 아키텍처 컴플라이언스 (v0.30 추가)
- [ ] 구현이 ARCHITECTURE.md 기술 스택을 따르는가
- [ ] 라이브러리로 대체 가능한 손코딩이 없는가 (ARCHITECTURE.md 명시 라이브러리 대신 직접 구현한 부분)
- [ ] 신규 패턴/라이브러리 도입 시 ARCHITECTURE.md에 기록되었는가
- [ ] package.json 설치 목록과 ARCHITECTURE.md 기술 스택이 일치하는가

## 판정 기준

- **APPROVE**: 모든 항목 통과
- **REQUEST_CHANGES**: spec/아키텍처 위반(ARCHITECTURE.md 기술 결정 미준수 포함), 보안, silent failure
- 스타일 취향이나 네이밍으로 REQUEST_CHANGES 금지

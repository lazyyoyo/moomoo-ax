# SLC — Simple / Lovable / Complete

수평 슬라이스(이번 제품 버전 후보)가 **그대로 출시 가능한 단위**인지 검증하는 3축 게이트.

## 세 축의 정의

| 축 | 질문 | 통과 신호 | 실패 신호 |
|---|---|---|---|
| **Simple** | "스코프가 최소한인가?" | Story 2~4개로 끝남, 비범위 명시 | Story 6개 이상 / "이것도 필요할 듯" 추가 |
| **Lovable** | "사용자가 쓰고 싶어하는 품질인가?" | 각 Story가 모달/페이지 단위로 쾌적하게 동작 | 텍스트만, 표지 없음, 로딩 인디케이터 없음 |
| **Complete** | "JTBD를 독립 충족하는가?" | 슬라이스만 배포해도 JTBD 한 줄이 성립 | "추가만 되고 삭제 안 됨", "조회만 되고 수정 안 됨" |

## 체크 예시

| 후보 슬라이스 | Simple | Lovable | Complete | 판정 |
|---|---|---|---|---|
| 책 추가 + 수정 + 삭제 | ○ | ○ | ○ | 통과 |
| 책 추가만 | ○ | ○ | ✗ (서재 관리 불완전) | 실패 → Story 추가 |
| 텍스트만으로 책 추가 | ○ | ✗ (표지 없으면 쓰기 싫음) | ○ | 실패 → 표지 검색 추가 |
| 로그인 + 회원가입 + 비번찾기 + 프로필 + 알림 | ✗ (스코프 폭발) | ○ | ○ | 실패 → 제품 버전 쪼개기 |
| 마이페이지(프로필+테마+비번) | ○ | ○ | ○ | 통과 (rubato v1.7.0) |

## 패치 모음 버전에서의 적용

패치 모음(예: rubato v1.5.1 — 안정성 회복)도 SLC를 통과해야 한다.

- **Simple** — 묶인 패치가 한 JTBD ("안정적으로 사용한다") 안에 든다. 무관한 신규 기능이 끼면 실패.
- **Lovable** — 사용자가 체감할 정도의 묶음 (개별 패치는 작아도 누적 효과로 "쾌적해졌다"). 내부 리팩터링만 모은 버전은 Lovable 실패.
- **Complete** — 묶인 패치들이 합쳐져 "안정성"이라는 JTBD를 일정 수준 종결시킴. 일부 알려진 버그가 남으면 실패 → 다음 패치 버전으로 분리.

## 실패 시 판단 기준

| SLC 실패 축 | 1차 처방 | 2차 처방 |
|---|---|---|
| Simple 실패 (스코프 폭발) | 슬라이스를 더 작게 (Story 빼기) | Story를 더 작게 쪼개거나 일부를 §비범위로 이동 |
| Lovable 실패 | 부족한 디테일을 Story에 추가 (검색·이미지·피드백 등) | 디테일이 다른 JTBD를 부른다면 다음 버전으로 |
| Complete 실패 | 누락된 Story를 슬라이스에 추가 | 추가 Story가 Simple을 깨면 → JTBD 자체를 좁혀서 재정의 |

> **원칙**: SLC 통과 못 하면 Phase A를 재진입한다. "일단 시작" 금지.

## scope.md §SLC 체크 형식

각 축을 한 줄 근거로 명시:

```markdown
## SLC 체크
- Simple: 3 Story로 끝. 계정 삭제·FAQ는 §비범위.
- Lovable: 각 Story가 모달/페이지 단위로 쾌적하게 동작 (인라인 검증·성공 화면 포함).
- Complete: 3 Story가 합쳐져 "마이페이지" 경험 독립 완결.
```

# 동적 체크리스트 생성 규칙 (CheckEval)

게이트 ④ Judge 체크리스트의 생성 규칙. UX 플로우 + DS를 입력으로 **Yes/No 이진 체크리스트**를 자동 생성한다.

> 점수(1~5)가 아니라 Yes/No. "디자인 완성도"를 분해하면 이진 질문들의 합이다.

## 체크리스트 2종

### 고정 루브릭 (항상 적용)

| # | 질문 | 검증 |
|---|---|---|
| F1 | DS 토큰만 사용했는가? (하드코딩 색상/간격 0건) | 게이트 ①에서 코드로 검증 |
| F2 | 레이아웃 규칙(max-width, header, spacing)을 준수했는가? | 게이트 ②에서 코드로 검증 |
| F3 | 모든 화면에 loading 상태가 있는가? | AI Judge |
| F4 | 모든 화면에 error 상태가 있는가? | AI Judge |
| F5 | 모든 화면에 empty 상태가 있는가? | AI Judge |
| F6 | 브랜드 색상이 일관되게 적용됐는가? | AI Judge |
| F7 | 타이포그래피 위계가 3단계 이상인가? | AI Judge |
| F8 | slop 패턴이 없는가? (그라디언트 남발, 과도한 그림자 등) | AI Judge |

### 동적 루브릭 (UX 기반 자동 생성)

UX 플로우의 각 화면/인터랙션에서 생성. **모든 항목은 Yes/No로 답할 수 있어야 한다.**

**생성 입력:**
- `flows/{기능명}.md` — 화면 목록 + 상태 변형 + 엣지 케이스
- `versions/vX.Y.Z/scope.md` §Story Map — Story별 기능 정의
- `references/domain-checklists.md` — 해당 도메인 섹션

**생성 규칙:**
1. flows의 각 화면에 대해 "이 화면의 핵심 인터랙션이 구현됐는가?" 형태의 질문 생성
2. 각 엣지 케이스에 대해 "이 엣지 케이스가 처리됐는가?" 형태의 질문 생성
3. domain-checklists.md에서 해당 도메인 항목을 Yes/No로 변환
4. 레퍼런스 매핑이 있으면 "레퍼런스의 {참고 포인트}가 반영됐는가?" 질문 추가

**예시 (독서 기록 기능):**
```
D1. 책 추가 화면에서 ISBN 스캔 → 자동 입력이 동작하는가? (Yes/No)
D2. 검색 결과 0건일 때 empty 상태가 표시되는가? (Yes/No)
D3. 네트워크 오류 시 재시도 UI가 있는가? (Yes/No)
D4. 중복 등록 시 경고가 표시되는가? (Yes/No)
D5. [레퍼런스] Goodreads의 shelf 이동 패턴이 반영됐는가? (Yes/No)
```

## 판정

- 고정 + 동적 전체에서 **No가 0건** → PASS
- **No가 1건 이상** → FAIL + No 항목 → 수정 지시서 자동 변환

## 실패 → 재작업 지시서 변환

No 항목을 수정 지시서로 자동 변환:

```
No: "모든 화면에 empty 상태가 있는가?"
→ 수정 지시서: "다음 화면에 empty 상태를 추가하라: [검색 결과, 서재 목록]"

No: "레퍼런스의 카드 hover elevation이 반영됐는가?"
→ 수정 지시서: "ProfileCard 컴포넌트에 hover 시 elevation 변화를 추가하라 (레퍼런스 B 참조)"
```

design-builder에 수정 지시서를 전달 → 재빌드 → 게이트 재실행.

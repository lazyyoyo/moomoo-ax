# 영문 폰트 카탈로그

Figma Resource Library의 폰트 가이드를 기반으로 정리.
각 폰트의 성격, 적합 도메인, 페어링 추천을 포함.

> 폰트 선택 프로토콜:
> 1. Brand에 지정된 폰트가 있으면 → 그것을 사용
> 2. 없으면 → 이 카탈로그에서 도메인 + 톤앤매너로 검색
> 3. 한국어 프로젝트면 → `korean-fonts.md` + `font-pairing-kr.md`와 교차 참조
> 4. 추가 탐색 필요 시 → Google Fonts API 사용

## Sans Serif — 범용/UI 최적화

### Inter
- **성격**: 중립적, 현대적, 기능적
- **특징**: 화면 최적화 설계, 다양한 weight, variable font 지원
- **Best for**: UI/UX, SaaS, 대시보드, 기술 문서
- **페어링**: + Krub (body에 약간의 개성), + Source Serif 4 (대비 효과)
- **주의**: 너무 많이 쓰여서 차별화가 어려움. 대안이 있으면 대안 우선 고려

### Open Sans
- **성격**: 깨끗하고, 단순하고, 친근한
- **특징**: Steve Matteson 디자인, 웹/모바일 최적화, 직립형 형태로 가독성 우수, 다국어 지원
- **Best for**: 범용 웹사이트, 기업 사이트, 모바일 인터페이스
- **페어링**: + Roboto Serif (본문), + Lora (에디토리얼)

### DM Sans
- **성격**: 기하학적(geometric), 미니멀, 부드러운 터미널
- **특징**: Montserrat의 웹 친화적 대안, 선명하고 현대적
- **Best for**: 웹사이트, 앱, 뉴스레터, UI 인터페이스
- **페어링**: + Instrument Serif (대비), + DM Serif Display (동일 패밀리)

### Plus Jakarta Sans
- **성격**: 친근하면서도 전문적, 현대적 geometric
- **특징**: 가독성 우수, 다양한 weight
- **Best for**: UI/UX, 모바일 앱, 디지털 브랜딩, 현대적 웹사이트
- **페어링**: + Space Mono (테크 느낌), + Instrument Serif

### Work Sans
- **성격**: 기하학적, 화면 최적화
- **특징**: 중간 weight는 본문용, 극단 weight는 디스플레이용, 큰 x-height
- **Best for**: 웹 디자인, 앱 인터페이스, 미니멀 디자인, 대시보드
- **페어링**: + Domine (serif 대비), + IBM Plex Mono (코드)

### Manrope
- **성격**: 깨끗하고 다목적, 미묘한 획 변조
- **특징**: 현대적 터치, 플랫폼과 크기에 걸쳐 우수한 가독성
- **Best for**: 브랜딩, 로고, 웹사이트, 헤드라인, 인터페이스, 본문
- **페어링**: + EB Garamond (클래식 대비)

### Poppins
- **성격**: 깨끗한 라인, 약간 응축, 친근하고 접근적
- **특징**: 미니멀 디자인, 다국어 지원
- **Best for**: 본문, 웹사이트, 브랜딩, 모바일 앱
- **페어링**: + Playfair Display (헤드라인), + Lora (에디토리얼)

### Rubik
- **성격**: 약간 둥근 모서리, 친근한
- **특징**: 5가지 weight + Roman/Italic, 오픈소스
- **Best for**: 헤더/타이틀/로고 (bold), 본문 (regular)
- **페어링**: + Karla (본문), + Roboto Mono (코드)

### Encode Sans
- **성격**: 약간 기술적 에지, 정돈된 외관
- **특징**: 일관된 letterform, 신중한 spacing
- **Best for**: 대시보드, 테크 웹사이트, 데이터 시각화, 앱
- **페어링**: + Source Serif 4 (대비)

### Epilogue
- **성격**: 깨끗하고 다용도, 열린 spacing, 안정적 구조
- **특징**: Lato나 Helvetica의 대안, 콘텐츠가 리드하고 타이포는 뒤로 물러남
- **Best for**: 콘텐츠 중심 프로젝트, 에디토리얼
- **페어링**: + serif 계열 헤드라인 폰트

### Be Vietnam Pro
- **성격**: 디지털 읽기에 특화, 넉넉한 spacing
- **특징**: Inter나 Roboto의 대안, 깨끗하고 구조적
- **Best for**: 대시보드, 웹사이트, 앱, 제품 인터페이스
- **페어링**: + 장식적(decorative) 폰트와 대비

### Cabin
- **성격**: 기하학적 정밀함 + 부드러운 곡선, 친근한 톤
- **특징**: Edward Johnston의 서체에서 영감, 화면 최적화
- **Best for**: 웹사이트, 프레젠테이션, UI, 따뜻한 브랜딩
- **페어링**: + serif 계열

### Inclusive Sans
- **성격**: 포용적, 접근성 우선
- **특징**: 열린 카운터, 명확한 글자 구분, 시각 장애/인지 장애 고려
- **Best for**: 접근성 중심 웹사이트, 교육 자료, UI
- **페어링**: + 가독성 높은 serif 폰트

### Instrument Sans
- **성격**: 깨끗하고 현대적, 다용도, 중립적
- **특징**: 균형잡힌 비율
- **Best for**: UI/UX, 브랜딩, 본문
- **페어링**: + Instrument Serif (동일 패밀리)

## Sans Serif — 디스플레이/임팩트

### Raleway
- **성격**: 우아한 sans serif, 기하학적 + 인간적 영향
- **특징**: 원래 얇은 단일 weight에서 9가지로 확장
- **Best for**: 디스플레이, 브랜딩, 헤드라인
- **페어링**: + Merriweather (본문, 축약된 letterform으로 편안한 읽기)

### Oswald
- **성격**: 굵고 응축된, 주의를 끄는
- **특징**: 헤더와 네비게이션에 강한 존재감
- **Best for**: 헤드라인, 네비게이션
- **페어링**: + Source Serif 4 (본문)

### Biryani
- **성격**: 굵은 sans serif, 좁은 letterform, 무겁지 않은 강한 존재감
- **특징**: 헤드라인/디스플레이용 설계
- **Best for**: 헤드라인, 배너, 제목
- **페어링**: + 가벼운 sans serif 본문

## Serif — 권위/클래식

### EB Garamond
- **성격**: Claude Garamond의 16세기 서체를 현대 디지털로 부활
- **특징**: 시대초월적 우아함
- **Best for**: 에디토리얼, 장문 콘텐츠, 학술
- **페어링**: + Manrope (현대적 대비)

### Libre Baskerville
- **성격**: 균형잡힌 비율, 뚜렷한 serif, 권위와 접근성의 균형
- **특징**: 긴 텍스트에서도 편안한 읽기 경험
- **Best for**: 뉴스 사이트, 온라인 출판, 전문 웹사이트
- **페어링**: + sans serif 네비게이션 폰트

### PT Serif
- **성격**: 다용도, 높은 가독성, 다국어 지원
- **특징**: 명확하고 균형잡힌 letterform, 적당한 serif
- **Best for**: 텍스트가 많은 웹사이트, 온라인 출판, 다국어 프로젝트
- **페어링**: + PT Sans (동일 패밀리)

### Roboto Serif
- **성격**: 현대적, 친근한, 열린 곡선 + 잘 정의된 serif
- **특징**: Variable font, Roboto 패밀리의 serif 버전
- **Best for**: 웹사이트/앱 본문, 에디토리얼 콘텐츠
- **페어링**: + Roboto (동일 패밀리)

### Instrument Serif
- **성격**: 클래식 serif + 현대적 미학
- **특징**: 우아함과 가독성의 균형
- **Best for**: 에디토리얼, 장문 콘텐츠, 서적
- **페어링**: + Instrument Sans (동일 패밀리)

## Serif — 디스플레이/럭셔리

### Bodoni Moda
- **성격**: 하이패션, 극적 대비, 세련됨
- **특징**: 극단적 대비 + 우아한 unbracket hairline, 디스플레이 전용
- **Best for**: 패션 브랜딩, 에디토리얼 디자인, 럭셔리 웹사이트, 헤드라인
- **페어링**: + 가벼운 sans serif 본문
- **주의**: 본문(body text)에는 부적합

### Didot
- **성격**: 신고전주의, 두꺼운 세로획과 극도로 얇은 가로획
- **특징**: 패션/럭셔리 산업의 상징적 서체
- **Best for**: 패션, 럭셔리, 에디토리얼 헤드라인
- **페어링**: + 가벼운 sans serif
- **주의**: 작은 크기에서 가독성 문제

### Abril Fatface
- **성격**: 굵고 우아한 serif, 주목을 끄는
- **특징**: 시선을 잡는 헤드라인용
- **Best for**: 럭셔리 브랜드, 에디토리얼 레이아웃
- **페어링**: + Lato (깨끗한 본문)

## Monospace — 코드/데이터

### Space Mono
- **성격**: 레트로 테크 느낌
- **Best for**: 테크 브랜드, 코딩 환경, 디자인 포트폴리오
- **페어링**: + Plus Jakarta Sans (친근한 대비)

### Inconsolata
- **성격**: 클래식 모노스페이스, Consolas/Avenir에서 영감
- **Best for**: 코드 리스팅, 기술 문서
- **페어링**: + 본문용 sans serif

## Script / Display — 포인트 사용

### Grand Hotel
- **성격**: 우아한 필기체, 핸드라이팅 느낌
- **Best for**: 웨딩, 부티크 브랜딩, 라이프스타일
- **페어링**: + Lato (구조적 본문)
- **주의**: 헤드라인/로고에만 사용, 본문 절대 금지

### Chonburi
- **성격**: 플레이풀, 레트로, 빈티지 핸드레터링에서 영감
- **Best for**: 크리에이티브 프로젝트, 특색있는 헤드라인
- **페어링**: + 깨끗한 sans serif 본문

### Dongle
- **성격**: 매력적, 둥근, 한국어 의성어에서 유래한 이름
- **특징**: 원래 한글용으로 설계, Latin 포함
- **Best for**: 웹사이트, 소셜미디어, 교육 자료, 포스터
- **페어링**: + 정돈된 sans serif

## 폰트 선택 의사결정 트리

```
프로젝트 도메인은?
├─ SaaS/테크/대시보드
│  ├─ 중립적: Work Sans, Encode Sans, Be Vietnam Pro
│  ├─ 친근한: Plus Jakarta Sans, Cabin
│  └─ 접근성 우선: Inclusive Sans
├─ 이커머스
│  ├─ 럭셔리: Bodoni Moda(제목) + 가벼운 sans(본문)
│  ├─ 모던: DM Sans, Manrope
│  └─ 친근한: Rubik, Poppins
├─ 콘텐츠/에디토리얼/리딩
│  ├─ 본문 serif: EB Garamond, Libre Baskerville, PT Serif
│  ├─ 본문 sans: Epilogue, Open Sans
│  └─ 제목: Abril Fatface, Instrument Serif
├─ 랜딩페이지/마케팅
│  ├─ 임팩트: Oswald(제목) + Source Serif 4(본문)
│  ├─ 세련된: Raleway(제목) + Merriweather(본문)
│  └─ 친근한: Rubik(제목) + Open Sans(본문)
└─ 크리에이티브/포트폴리오
   ├─ 미니멀: Manrope, Epilogue
   ├─ 레트로 테크: Space Mono + Plus Jakarta Sans
   └─ 실험적: 도메인에 맞는 display 폰트 선택
```

## 폰트 페어링 원칙

1. **대비(Contrast)**: 제목과 본문이 충분히 달라야 시각적 위계가 생김. Serif + Sans Serif이 가장 안전한 조합.
2. **조화(Harmony)**: 같은 패밀리(Instrument Sans + Instrument Serif) 또는 같은 디자인 철학의 폰트끼리.
3. **최대 3종**: 한 프로젝트에 3개 이상의 폰트 패밀리를 쓰지 마라. 2개가 이상적.
4. **역할 분명히**: 각 폰트의 역할(제목, 본문, 코드/데이터, 액센트)을 명확히 정해라.
5. **실제 콘텐츠로 테스트**: Lorem ipsum이 아닌 실제 텍스트로 다양한 크기와 weight를 확인.

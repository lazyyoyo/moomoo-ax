# 레이아웃 패턴 목록

매 프로젝트/phase마다 다른 레이아웃 패턴을 선택하여 다양성을 확보한다.
이전 phase에서 사용한 패턴은 다음 phase에서 반복하지 않는다.

---

## 그리드 기반

### Bento Grid
- **설명**: 다양한 크기의 카드/블록이 모듈형으로 배치된 그리드. 일본 도시락 박스에서 영감.
- **특징**: 대형 카드 2~3개 + 소형 보조 카드, 과장된 둥근 모서리, 각 타일 내 마이크로 인터랙션
- **적합**: 피처 섹션, 대시보드, 포트폴리오, 서비스 오버뷰, 제품 카테고리
- **주의**: 모든 카드가 동일 크기면 스프레드시트처럼 보임. 시각적 위계 필수.
- **CSS 힌트**: `display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));` + 특정 카드에 `grid-column: span 2;`

### Standard Grid (12-column)
- **설명**: 전통적인 12컬럼 그리드 시스템
- **특징**: 가장 안정적이고 예측 가능한 레이아웃
- **적합**: 기업 사이트, 뉴스, 이커머스 상품 목록
- **주의**: 안전하지만 지루할 수 있음. 컬럼 폭 변화나 offset으로 변화를 줄 것.

### Masonry (Pinterest 스타일)
- **설명**: 높이가 다른 카드들이 벽돌 쌓듯 채워지는 레이아웃
- **특징**: 시각적으로 역동적, 공간 효율 높음
- **적합**: 이미지 갤러리, 포트폴리오, 소셜 피드
- **CSS 힌트**: CSS `columns` 또는 Masonry.js, 2026년부터 CSS native masonry 지원 확대

## 비대칭/실험적

### Broken Grid
- **설명**: 의도적으로 그리드를 깨뜨리고 요소를 겹치거나 어긋나게 배치
- **특징**: 역동적, 예술적, 시선을 끄는
- **적합**: 크리에이티브 에이전시, 포트폴리오, 패션, 문화
- **주의**: 과하면 혼란. 시선 흐름은 여전히 제어되어야 함.
- **CSS 힌트**: `position: relative;` + `transform: translateX(-20px);` 또는 `grid` + 의도적 overlap

### Split Screen
- **설명**: 화면을 좌우 또는 상하 50:50으로 분할
- **특징**: 두 가지 메시지/경로를 동시에 제시, 강한 대비 효과
- **적합**: 랜딩페이지(두 가지 선택지), 비교 페이지, Before/After
- **변형**: 40:60 또는 30:70 비율로 비대칭 분할

### Asymmetric Layout
- **설명**: 요소를 의도적으로 불균형하게 배치하되 시각적 균형은 유지
- **특징**: 큰 이미지 + 작은 텍스트 블록, 또는 넓은 여백 + 밀집 콘텐츠
- **적합**: 에디토리얼, 럭셔리, 스토리텔링
- **주의**: "의도된 비대칭"이어야지 "실수처럼 보이는 비대칭"이면 안 됨

### Diagonal / Angular
- **설명**: 대각선이나 기울어진 요소로 역동성을 만드는 레이아웃
- **특징**: 에너지, 움직임, 진보의 느낌
- **적합**: 스포츠, 테크, 스타트업, 이벤트
- **CSS 힌트**: `clip-path: polygon(...)` 또는 `transform: skewY(-3deg);`

## 콘텐츠 중심

### Editorial / Magazine
- **설명**: 잡지 편집 레이아웃에서 영감. 텍스트와 이미지의 정교한 조합.
- **특징**: 큰 헤드라인 + 멀티컬럼 본문 + 풀블리드 이미지 + 인용구 삽입
- **적합**: 블로그, 뉴스, 매거진, 장문 콘텐츠
- **CSS 힌트**: `column-count: 2;` + `break-inside: avoid;` + 큰 이미지에 `grid-column: 1 / -1;`

### Single Column (좁은 본문)
- **설명**: 하나의 좁은 컬럼에 텍스트가 흐르는 레이아웃. 최대 650~750px 폭.
- **특징**: 읽기에 최적화, 집중력 유지, 미니멀
- **적합**: 블로그 포스트, 에세이, 문서, 이메일
- **주의**: 사이드바나 보조 요소 없이 콘텐츠만으로 승부. 타이포그래피가 핵심.

### Card-Based
- **설명**: 콘텐츠를 독립적인 카드 단위로 나누어 배치
- **특징**: 각 카드가 자기완결적 정보 단위, 반응형에 유리
- **적합**: 뉴스 피드, 제품 목록, 대시보드, SNS
- **주의**: Bento Grid와 다른 점은 카드 크기가 비교적 균일하다는 것

## 전체 화면/몰입형

### Full-Bleed Hero
- **설명**: 화면 전체를 채우는 히어로 이미지/비디오 + 오버레이 텍스트
- **특징**: 강렬한 첫인상, 브랜드 메시지 집중
- **적합**: 랜딩페이지, 제품 런칭, 포트폴리오 메인
- **주의**: 히어로 아래 콘텐츠로의 전환이 자연스러워야 함. 스크롤 유도 요소 필요.

### Scroll Storytelling (Long-form Scroll)
- **설명**: 스크롤에 따라 콘텐츠가 순차적으로 등장/변화하는 몰입형 레이아웃
- **특징**: 패럴랙스, 스크롤 트리거 애니메이션, 섹션 간 전환 효과
- **적합**: 제품 소개, 브랜드 스토리, 연례 보고서, 인터랙티브 에세이
- **CSS/JS**: Intersection Observer API + CSS transitions, 또는 GSAP ScrollTrigger

### Sticky Sidebar + Scrolling Content
- **설명**: 한쪽은 고정(sticky), 다른 쪽은 스크롤되는 레이아웃
- **특징**: 네비게이션/TOC/이미지를 고정하고 콘텐츠만 스크롤
- **적합**: 문서, API 레퍼런스, 비교 페이지, 프라이싱
- **CSS 힌트**: `position: sticky; top: 0;`

## 특수 패턴

### Z-Pattern
- **설명**: 사람의 시선이 Z자 형태로 이동하는 것을 활용한 배치
- **특징**: 좌상단 → 우상단 → 좌하단 → 우하단 순서로 중요 요소 배치
- **적합**: 랜딩페이지, 간단한 정보 페이지
- **팁**: 로고(좌상단) → CTA(우상단) → 가치제안(좌하단) → CTA(우하단)

### F-Pattern
- **설명**: 사람의 시선이 F자 형태로 스캔하는 것을 활용한 배치
- **특징**: 상단 가로 → 중간 가로 → 좌측 세로 스캔
- **적합**: 텍스트 중심 페이지, 검색 결과, 기사 목록

### Hub & Spoke
- **설명**: 중앙 요소(허브)에서 방사형으로 연결되는 하위 요소들
- **특징**: 관계성 표현, 네비게이션 구조 시각화
- **적합**: 서비스 오버뷰, 기능 연결 다이어그램, 메인 대시보드

## 레이아웃 선택 의사결정 트리

```
콘텐츠 유형은?
├─ 다양한 유형의 카드/블록
│  ├─ 시각적 위계가 필요 → Bento Grid
│  ├─ 균일한 카드 → Card-Based
│  └─ 이미지 중심, 높이 가변 → Masonry
├─ 장문 텍스트
│  ├─ 읽기 집중 → Single Column
│  ├─ 매거진 느낌 → Editorial / Magazine
│  └─ 참조 자료 + 본문 → Sticky Sidebar
├─ 스토리텔링/내러티브
│  ├─ 몰입형 → Scroll Storytelling
│  ├─ 강렬한 첫인상 → Full-Bleed Hero
│  └─ 두 가지 경로 → Split Screen
├─ 실험적/크리에이티브
│  ├─ 역동적 → Broken Grid 또는 Diagonal
│  └─ 세련된 불균형 → Asymmetric Layout
└─ 정보 구조화
   ├─ 비교/선택 → Split Screen
   ├─ 시선 유도 → Z-Pattern 또는 F-Pattern
   └─ 관계 표현 → Hub & Spoke
```

## 반복 회피 규칙

한 프로젝트 내에서 동일한 레이아웃 패턴의 반복을 최소화한다:

1. **히어로 섹션**과 **피처 섹션**은 반드시 다른 패턴 사용
2. 연속된 두 섹션이 같은 그리드 구조면 배경색/여백/방향을 변화시켜 리듬 생성
3. 한 페이지에 Card-Based 섹션이 2개 이상이면 하나는 다른 패턴으로 교체
4. 프로젝트 전체에서 "기본 패턴"은 1개만 허용, 나머지 섹션은 변주

# 도메인별 UI/UX 체크리스트

도메인에 따라 UI/UX에서 중요한 요소가 다르다. Plan 단계에서 도메인을 판별한 후 해당 섹션을 로드하여 체크리스트로 활용한다. UX Review 시에도 이 체크리스트를 기준으로 검증한다.

---

## SaaS / 웹 애플리케이션

### 온보딩
- [ ] 첫 방문 사용자를 위한 명확한 진입 경로
- [ ] 기능 설명이 아닌 가치(value) 중심의 첫 화면
- [ ] 단계별 가이드 또는 프로그레시브 디스클로저
- [ ] 빈 상태(Empty state) 디자인 — 데이터 없을 때 무엇을 보여줄지
- [ ] 스킬 적응형 온보딩: 초보자는 단계별, 숙련자는 스킵 가능

### 네비게이션
- [ ] 일관된 사이드바 또는 탑바 네비게이션
- [ ] 현재 위치 표시 (active state)
- [ ] 깊이 3단계 이상의 네비게이션 금지
- [ ] 검색 기능 접근성 (Cmd+K 단축키 고려)
- [ ] 브레드크럼 (깊은 구조일 경우)

### 데이터 표시
- [ ] 데이터 테이블: 정렬, 필터, 페이지네이션
- [ ] 로딩 상태: 스켈레톤 UI 또는 의미있는 로딩 인디케이터
- [ ] 에러 상태: 무엇이 잘못되었는지 + 어떻게 해결할 수 있는지
- [ ] 대시보드: 핵심 지표를 최상단에, 세부 사항은 드릴다운
- [ ] 차트/그래프: 컬러 접근성 확인 (색각 이상자 고려)

### 다크 모드
- [ ] 다크 모드 지원 (2026년 SaaS에서는 거의 필수)
- [ ] 순수 검정(#000) 대신 다크 그레이 사용
- [ ] 모든 컴포넌트 상태(hover, focus, disabled, error)가 다크 모드에서 작동
- [ ] 시스템 설정 연동 + 수동 토글

### 일반
- [ ] 실행 취소/되돌리기 가능한 액션
- [ ] 키보드 단축키 (파워 유저용)
- [ ] 설정 페이지의 명확한 구조
- [ ] 알림/토스트 메시지의 일관된 패턴

## 이커머스 / 리테일

### 상품 탐색
- [ ] 명확한 카테고리 구조
- [ ] 필터/정렬: 가격, 인기도, 최신순, 리뷰 점수
- [ ] 상품 카드: 이미지 품질, 가격, 핵심 정보 노출
- [ ] 그리드 뷰 / 리스트 뷰 전환
- [ ] 무한 스크롤 또는 페이지네이션 (둘 중 하나 선택, 혼합 금지)

### 상품 상세
- [ ] 고해상도 이미지 + 줌 기능
- [ ] 명확한 가격 표시 + 할인 정보
- [ ] CTA 버튼("장바구니 담기")의 항상 보이는 위치
- [ ] 사이즈/옵션 선택의 직관적 UI
- [ ] 리뷰/평점 섹션
- [ ] 관련 상품 추천

### 결제 플로우
- [ ] 단계 표시 (1/3, 2/3, 3/3)
- [ ] 폼 필드 최소화
- [ ] 실시간 유효성 검사
- [ ] 주문 요약 항상 보이게
- [ ] 게스트 결제 옵션
- [ ] 신뢰 신호 (보안 배지, 환불 정책)

### 모바일
- [ ] 엄지 영역(thumb zone)에 주요 CTA 배치
- [ ] 스와이프로 상품 이미지 넘기기
- [ ] 고정 하단 바 (장바구니, 검색)

## 콘텐츠 / 블로그 / 리딩 서비스

### 타이포그래피
- [ ] 본문 폰트 크기 최소 16px (모바일), 18px+ (데스크톱)
- [ ] 줄간격(line-height): 한글 1.6~1.8em, 영문 1.5~1.7em
- [ ] 한 줄 최대 글자 수: 한글 35~45자, 영문 50~75자
- [ ] 문단 간격이 충분한가 (줄간격의 1.5배 이상)
- [ ] 제목-본문-캡션의 명확한 위계 (최소 3단계)

### 읽기 경험
- [ ] 읽기 집중 모드 (사이드바/광고 없는 깨끗한 레이아웃)
- [ ] 스크롤 진행률 표시 (선택)
- [ ] 목차 (TOC) — 긴 글에서 필수
- [ ] 다크 모드 / 세피아 모드 (읽기 편의)
- [ ] 글꼴 크기 조절 기능 (선택)

### 콘텐츠 구조
- [ ] 스캔 가능한 헤딩 구조 (H1 → H2 → H3)
- [ ] 이미지/인용구/코드블록 등 시각적 브레이크
- [ ] 관련 콘텐츠 추천
- [ ] 공유 버튼 (비침습적 위치)
- [ ] 태그/카테고리 네비게이션

### 한국어 특화
- [ ] word-break: keep-all 적용 (한국어 단어 단위 줄바꿈)
- [ ] 한글 폰트의 자간(letter-spacing) 미세 조정
- [ ] 한글-영문-숫자 혼용 시 가독성 확인
- [ ] 세로쓰기 지원 여부 (특수 케이스)

## 랜딩페이지 / 마케팅

### Above the Fold
- [ ] 핵심 가치 제안(Value Proposition)이 3초 내 파악 가능
- [ ] 하나의 명확한 CTA
- [ ] 히어로 이미지/비디오가 가치 제안을 보강
- [ ] 사회적 증거 (고객 수, 로고, 평점) — 빠르게 노출

### 스크롤 구조
- [ ] 문제 → 해결책 → 증거 → CTA 순서의 내러티브
- [ ] 각 섹션이 하나의 메시지만 전달
- [ ] 반복되는 CTA (스크롤 중간중간)
- [ ] 섹션 간 시각적 구분 (배경색 변화, 여백, 구분선)

### 신뢰 구축
- [ ] 고객 후기/케이스 스터디
- [ ] 미디어 노출 (As seen on...)
- [ ] 파트너/고객 로고 (로고 바)
- [ ] FAQ 섹션
- [ ] 보안/개인정보 보호 메시지

### 전환 최적화
- [ ] CTA 버튼의 시각적 돌출 (컬러 대비)
- [ ] 폼 필드 최소화 (이메일 하나로 시작)
- [ ] 마이크로 커밋먼트: 작은 액션 → 큰 전환으로 유도
- [ ] 긴급성/희소성 신호 (적절하게 사용 시)

### 성능
- [ ] LCP(Largest Contentful Paint) 2.5초 이내
- [ ] Above the fold 이미지 preload
- [ ] 스크롤 아래 콘텐츠 lazy loading

## 대시보드 / 어드민

### 정보 계층
- [ ] 가장 중요한 지표를 최상단에 (KPI 카드)
- [ ] 세부 정보는 드릴다운으로
- [ ] 정보 밀도 vs 여백의 균형 — 너무 빽빽하지도, 너무 비어있지도 않게
- [ ] 데이터 그룹핑: 관련 정보끼리 시각적 근접성

### 데이터 시각화
- [ ] 차트 유형이 데이터에 적합한가 (선 그래프 vs 막대 vs 파이)
- [ ] 컬러 팔레트가 접근성을 고려했는가 (색각 이상)
- [ ] 축 레이블, 범례가 명확한가
- [ ] 인터랙티브: hover 시 상세 데이터 표시
- [ ] 기간 필터 (오늘, 주간, 월간, 커스텀)

### 테이블
- [ ] 정렬 가능한 컬럼 헤더
- [ ] 행 hover 하이라이트
- [ ] 대량 데이터: 가상 스크롤 또는 페이지네이션
- [ ] 컬럼 고정 (sticky header, sticky first column)
- [ ] 행 선택 + 벌크 액션

### 네비게이션
- [ ] 접이식 사이드바 (공간 절약)
- [ ] 빠른 접근: 즐겨찾기/핀 기능
- [ ] 글로벌 검색
- [ ] 알림 센터

### 다크 모드
- [ ] 대시보드에서 다크 모드는 거의 필수
- [ ] 데이터 시각화 컬러가 다크 배경에서 작동하는지 확인
- [ ] 차트의 그리드 라인, 축 색상 조정

## 모바일 앱 / 모바일 웹

### 터치 인터페이스
- [ ] 터치 타겟 최소 44x44px
- [ ] 엄지 도달 영역에 주요 액션 배치
- [ ] 스와이프 제스처 활용 (삭제, 아카이브 등)
- [ ] 하단 네비게이션 바 (주요 탭)

### 성능
- [ ] 오프라인 기본 동작 고려
- [ ] 이미지 최적화 (WebP, 적절한 해상도)
- [ ] 폰트 로딩 최적화 (font-display: swap)
- [ ] 첫 화면 로딩 3초 이내

### 반응형
- [ ] 세로(portrait) 모드 최적화 우선
- [ ] 가로(landscape) 모드에서도 레이아웃 깨지지 않음
- [ ] 노치/다이나믹 아일랜드 영역 고려 (safe-area-inset)
- [ ] 하단 제스처 바 영역 고려

### 일반
- [ ] 풀-투-리프레시 패턴
- [ ] 로딩 상태에서의 스켈레톤 UI
- [ ] 에러 상태에서의 재시도 옵션
- [ ] 뒤로 가기 동작의 예측 가능성

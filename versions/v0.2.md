# v0.2 — 루프 엔진 + CPS 작성 실험 + 관측 인프라

## 목표

1. 범용 루프 엔진(loop.py + judge.py)을 만든다
2. labs/cps-gen/에서 "CPS 생성 스크립트"를 루프로 최적화
3. 로그를 Supabase에 쌓고 Vercel 대시보드에서 확인

## 왜 CPS부터?

- PRD 작성 루프를 만들려면 input으로 CPS가 필요 → CPS가 없음
- CPS는 PRD보다 단순 → 루프 엔진 검증 첫 타겟으로 적합
- input: 러프한 아이디어 (이건 있음) → output: 구조화된 CPS
- v0.2에서 만든 CPS를 v0.3 PRD 루프의 input으로 사용

## 핵심 구조

### auto-research 매핑

| auto-research | moomoo-ax | 누가 관리 |
|---------------|----------|----------|
| program.md | `program.md` — 오너 규칙 (톤, 구조, 제약) | 오너 |
| train.py | `script.md` — 생성 스크립트 (AI가 루프에서 개선) | AI |
| prepare.py | `rubric.yml` — 평가 체크리스트 | 오너 (루프 안 불변) |
| results.tsv | `logs/` + Supabase | 자동 |

### 루프 흐름

```
script.md로 산출물 생성 → rubric으로 산출물 eval → 점수
  → 점수 > best → keep (script.md 저장 + best 갱신)
  → 점수 ≤ best → discard (실패 항목 → script.md 수정 피드백)
→ 종료: 점수 ≥ 임계값 or max iteration
```

## 첫 실험: cps-gen

### input

러프한 아이디어. 예: "ax-loop 실행 로그를 Supabase에 쌓고, 프로젝트별/유저별 점수 추이를 Vercel 대시보드에서 보여주는 것"

### program.md (오너 규칙)

```markdown
# CPS 생성 규칙

## CPS 구조
- **Context**: 현재 상황, 배경, 관련 시스템/유저
- **Problem**: 해결하려는 문제 (구체적, 측정 가능하면 좋음)
- **Solution**: 제안하는 해결책 (핵심 아이디어 + 범위)

## 톤
- 간결하고 명확
- 각 섹션 3~5문장
- "왜 이게 문제인지"가 드러나야 함

## 제약
- 마크다운 형식
- 500자 이내
```

### rubric.yml

```yaml
items:
  - question: "Context가 현재 상황을 구체적으로 설명하는가?"
    critical: true
  - question: "관련 시스템/유저가 명시되어 있는가?"
    critical: false
  - question: "Problem이 한 문장으로 명확히 정의되어 있는가?"
    critical: true
  - question: "문제의 영향(누가 어떻게 불편한지)이 드러나는가?"
    critical: true
  - question: "Solution이 문제에 대한 직접적 답인가?"
    critical: true
  - question: "Solution의 범위가 명확한가 (뭘 하고 뭘 안 하는지)?"
    critical: true
  - question: "Context → Problem → Solution 논리 흐름이 자연스러운가?"
    critical: false
  - question: "500자 이내인가?"
    critical: false
  - question: "불필요한 서론/배경 없이 바로 본론인가?"
    critical: false
```

### script.md (초안)

```markdown
# CPS 생성 스크립트

러프한 아이디어를 입력받아 CPS 문서를 생성한다.

## 절차
1. 아이디어에서 핵심 키워드/의도 추출
2. Context 작성: 현재 상황 + 관련 시스템/유저
3. Problem 작성: 해결하려는 문제 1문장 + 영향
4. Solution 작성: 핵심 해결책 + 범위
5. 논리 흐름 검증: C→P→S가 자연스러운지

## 출력 형식
마크다운. 500자 이내. C/P/S 3섹션.
```

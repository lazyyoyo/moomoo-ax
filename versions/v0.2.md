# v0.2 — Define 파이프라인 + 루프 엔진

## 목표

1. script.py 기반 루프 엔진 (실행 가능한 코드)
2. Define 파이프라인: 러프 아이디어 → Stage 1~3 자동 → PO 게이트 → Stage 4~5 자동 → PRD
3. Supabase 로그 + Vercel 대시보드

## 파이프라인

```
러프 아이디어
  → Stage 1: Seed Capture    → seed.md
  → Stage 2: JTBD Discovery  → jtbd.md
  → Stage 3: Problem Framing → problem-frame.md
  ── PO 게이트 (솔루션 선택 확정) ──
  → Stage 4: Scope Definition → scope.md
  → Stage 5: PRD Generation   → prd.md
```

Stage 1→2→3 자동. PO 확인 후 4→5 자동.

## 핵심 변경: script.md → script.py

| 이전 | 이후 |
|------|------|
| script.md (자연어 프롬프트) | script.py (실행 가능한 코드) |
| Claude가 해석해서 따름 (비결정적) | Python이 실행하면 산출물 나옴 (결정적) |
| 루프가 프롬프트 텍스트를 수정 | 루프가 코드(프롬프트, 후처리, 파싱)를 수정 |

## pipeline.py

```python
def run_define(idea: str, po_gate=True):
    """Define 파이프라인: idea → PRD"""
    # Stage 1
    seed = run_stage("seed-gen", idea)
    # Stage 2
    jtbd = run_stage("jtbd-gen", seed)
    # Stage 3
    problem = run_stage("problem-frame", seed + jtbd)
    
    if po_gate:
        # PO 확인 대기
        print("=== PO 게이트: problem-frame.md 확인 후 계속 ===")
        input("Enter to continue...")
    
    # Stage 4
    scope = run_stage("scope-gen", problem)
    # Stage 5
    prd = run_stage("prd-gen", seed + jtbd + problem + scope)
    
    return prd
```

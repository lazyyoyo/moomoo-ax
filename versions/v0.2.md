## v0.2 목표

**"단일 업무에 대해 점수가 올라가는 루프"를 1회라도 end-to-end로 돌린다.**

## 설계

### 루프 구조

```
input (CPS/지시) → LLM이 산출물 생성 → eval(루브릭 → 점수) → 판정
  → 점수 > best → keep (산출물 저장 + best 갱신)
  → 점수 ≤ best → discard (실패 항목 피드백 → 다음 반복)
→ 종료 조건: 점수 ≥ 임계값 or max iteration
```

### auto-research 매핑

| auto-research | v0.2 |
|---------------|------|
| program.md | `tasks/{task-name}/program.md` — 역할 + 규칙 |
| train.py (수정 대상) | `tasks/{task-name}/output/` — 산출물 파일 |
| prepare.py (eval) | `tasks/{task-name}/rubric.yml` — 체크리스트 |
| results.tsv | `tasks/{task-name}/logs/` — iteration별 JSON |

### loop.py — 오케스트레이터

```python
# 핵심 로직 (의사코드)
def run(task_dir, max_iter=10, threshold=0.85):
    program = read(task_dir / "program.md")
    rubric = read(task_dir / "rubric.yml")
    input_context = read(task_dir / "input/")
    
    best_score = 0
    best_output = None
    feedback = None
    
    for i in range(max_iter):
        # 1. 생성
        output = call_claude(program, input_context, feedback)
        save(task_dir / "output/", output)
        
        # 2. 평가
        score, failed_items = judge(rubric, output)
        
        # 3. 판정
        if score > best_score:
            best_score = score
            best_output = output
            verdict = "keep"
        else:
            verdict = "discard"
        
        # 4. 로그
        save_log(task_dir / "logs/", i, score, verdict)
        
        # 5. 종료 체크
        if score >= threshold:
            break
        
        # 6. 피드백 (다음 반복용)
        feedback = failed_items
    
    return best_output, best_score
```

핵심: subprocess.run으로 Claude 호출. tmux 없음. 최대한 단순.

### rubric.yml 형식

```yaml
items:
  - question: "문제 정의가 명확한가?"
    critical: true
  - question: "타겟 유저가 구체적으로 정의되어 있는가?"
    critical: true
  - question: "성공 지표가 측정 가능한가?"
    critical: false
  - question: "기술 제약이 명시되어 있는가?"
    critical: false
```

critical 항목이 하나라도 No이면 점수와 무관하게 실패.

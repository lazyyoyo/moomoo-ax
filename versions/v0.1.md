### 이슈

#### #1 tmux 세션 미생성 (severity: critical)

**증상**: `ax-workers` tmux 세션이 새로 안 열리고, 기존 터미널 세션에서 Claude가 실행됨.
**원인**: `ensure_session()`에서 `capture_output=True`로 tmux 에러가 삼켜짐. tmux 서버가 안 떠있는 상태에서 세션 생성 실패해도 감지 못함.
**수정**: worker.py에 에러 핸들링 추가 (커밋 86c1099). 세션/창 생성 실패 시 명확한 에러 반환.
**상태**: 코드 수정 완료, 재테스트 필요.

#### #2 .harness/ 디렉토리 미생성 (severity: critical)

**증상**: 대상 프로젝트(journal)에 `.harness/` 디렉토리가 생성되지 않음. 체크포인트, 로그 없음.
**원인 추정**: tmux 이슈(#1)로 워커가 정상 경로로 실행되지 않아 orchestrator의 루프 진입 전 실패했거나, gate 결과 파일 경로(`harness_dir()` 호출)까지 도달 못했을 가능성.
**상태**: #1 수정 후 재테스트로 확인 필요. 별도 버그일 수도 있음.
